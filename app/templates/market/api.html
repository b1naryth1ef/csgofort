Welcome to the CSGO Fort Market API Documentation! This contains everything you need to get started reading data from our public API endpoints. Keep in mind, this documentation was written for those with some prior knowledge of how to interact and use programatic web services.

Structure

Basics

The Fort Market API was intentionally structured to be helpful to outsider users. As such, most of the common tasks we expect people to need are easy through the exposed methods. If you think we've missed something, we're always happy to hear your input so feel free to contact us! The API resembles a REST-ish application, with our primary data structures being "Market Items" and "Market Item Price Points". The entire API is read-only, and thus all endpoints will only accept GET requests. All data returned is in standard JSON format, with datetimes in the standard ISO format.

Authentication/Limits

The API does not require you to authenticate or use API keys to access the endpoints, however we do impose some (rather generous) limits on users, based on IP. The basic structure is 6000 requests every 10 minutes, per IP. If you need more, you will have to contact us to get your IP whitelisted. API endpoints will return a 429 (Too Many Requests) upon hitting the limit. Keep in mind, we reserve the right to change the behavior of these limits at any time without contacting you.

Crediting

If you use the Fort Market API in any commercial or monetized application, you must give clear credit towards our API, and link directly to this API page. We also request any users that are not monetizing or profiting from their application link towards our API, however it is not required. Keep in mind we reserve the right to instantly terminate access rights to commercial users who do not give proper credit towards the API

Routes

market.csgofort.com/info

{
    "latest": 211,

    "total_listings": 74,
    "value": 12940
    "success": true,
}
This API returns some general information and statistics about the API, such as the total number of unique items stored (total_items), the total number of current listings (total_listings), and the current estimated market value (value). Latest represents the most recently added item.
